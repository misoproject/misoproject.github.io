Miso.Dataset

A Miso.Dataset also extends from Miso.DataView. All the methods available on a dataview will also be available on the dataset. 

new Miso.Dataset(options)

Params:

  • options - Optional arguments
    • url - A url to a location containing your data.
    • sync - Set to true to be able to bind to dataset changes. False by default.
    • jsonp - When fetching a url, set to true if call is jsonp request.
    • delimiter - If the data is delimited, set to the string that represents the delimiter.
    • string - Set to true if data is in the internal dataset format.
    • extract - Optional passthrough function for the data before it goes into the parser to extract the actual row containing values from a potentially larger set of data.
    • ready - Callback function that will be called once data is fetched. Will be called before success callback on fetch.
    • columns - A way to manually override column type detection. Expects an array of objects of the following structure: { name : 'columnname', type: 'columntype', ... (additional params required for type here.)}
    • comparator - A function to sort the data by. It will be sorted on fetch and on any successive addition.
    • deferred - A way to provide your own deferred object. It needs to follow the standard deferred signatures.
    • importer - The classname of any importer (passes through auto detection based on parameters. For example: Miso.Importers.Polling.
    • parser - The classname of any parser (passes through auto detection based on parameters. For example: Miso.Parsers.Delimited.
    • resetOnFetch - set to true if any subsequent fetches after first one should overwrite the current data.
    • uniqueAgainst - Set to a column name to check for duplication on subsequent fetches.
    • interval - Polling interval. Set to any value in milliseconds to enable polling on a url.

    • Returns:

    Miso.Dataset

    Description:

    Dataset Constructor

    Miso.Dataset Instance Methods

    In this section, ds will be considered an instance of Miso.Dataset. 
    ds.fetch(options)

    Params:

  • options - Optional Arguments
    • success - Success callback to be called when data is fetched. Context is the dataset.
    • error - Error callback to be called when data fetching failed.. Context is the dataset.

    • Returns:

    Deferred

    Description:

    Fetches the data and populates the dataset. Even when importing local objects from a var, this needs to be called. 
    ds.apply(data)

    Params:

  • data - Data to be applied (added or replaceing) to the dataset in whatever format the parser expects

    • Description:

    Applies the data to the dataset. It can add it blindly, replace all existing or add only unique rows based on a column. This is called in fetch and most likely will never need to be called. 
    ds.addColumn(column)

    Params:

  • column - Column options
    • name - Column name
    • type - Column type (string name of type)
    • format - Optiona. Only set if time type. The moment.js format describing the input dates.
    • id - Optional. Sets a custom column _id. We assign one by default.
    • data - Optional. A set of data. By default, set to an empty array.

    • Returns:

    Miso.Column

    Description:

    Creates a new column and adds it to the dataset. 
    ds.addColumns(columns)

    Params:

  • columns - Array of column objects

    • Description:

    Adds all columns. See ds.addColumn(column) for specification of properties. 
    ds.add(row, options)

    Params:

  • row - Object representing a new row
  • options - Optional Arguments
    • silent - Set to true to disable event firing

    • Description:

    Adds a row to the dataset. This will fire add and change events on a syncable dataset. 
    ds.remove(filter, options)

    Params:

  • filter - Can be one of two things: A row id, or a filter function that takes a row and returns true if that row should be removed or false otherwise.
  • options - Optional Arguments
    • silent - Set to true to disable event firing

    • Description:

    Removes a row from a dataset. This will fire remove and change events on a syncable dataset. 
    ds.update(filter, newProperties, options)

    Params:

  • filter - Can be one of two things: A row id, or a filter function that takes a row and returns true if that row should be updated or false otherwise.
  • newProperties - An object representing the values that are changing.
  • options - Optional Arguments
    • silent - Set to true to disable event firing

    • Description:

    Updates one or more rows in a dataset. This will fire update and change events on a syncable dataset. 
    ds.reset()

    Description:

    Resets the dataset by removing all the data from the coluns. Fires a reset event.

    Miso.DataView

    A Miso.DataView is an immutable version of dataset. It is the result of selecting a subset of the data using the ds.where call. If the dataset is syncing, this view will be updated when changes take place in the original dataset.

    A Miso.Dataset also extends from Miso.DataView. All the methods available on a dataview will also be available on the dataset. 

    new Miso.View(options)

    Params:

  • options - DataView options
    • parent - The parent dataset
    • filter - Filtering instructions:
      • columns - Optional. A filter for columns. A single or multiple column names.
      • rows - Optional. A filter for rows. A rowId or a filter function that takes in a row and returns true if it fits the criteria or not.

    • Description:

    Creates a subset of the original dataset. This view behaves like a regular dataset in that it can be further queried, but it cannot be modified. This should not be manually created as it is created for you when you call the appropriate methods in Miso.Dataset.

    Miso.Importer

    To specify a custom importer during dataset instantiation, just set the importer property to the class name of the importer you want. Some importers require custom properties. This section will document the properties you can set that will either cause this importer to be selected or will be required by it to continue.

    Built in Importers include:

    • Miso.Importers.Local
    • Miso.Importers.Remote
    • Miso.Importers.Polling
    • Miso.Importers.GoogleSpreadsheet

    An importer must implement the following interface:

    Miso.Importer.Local

    Required dataset instantiation property:

    • data - The local var object containing the data.

    Miso.Importer.Remote

    Required dataset instantiation property:

    • url - The url from which to fetch the data.
    • jsonp - If the url in question is a jsonp request, set this to true.

    Miso.Importer.Polling

    • interval - The interval at which each request will be made. This is an approximate interval because until a request returns, the next request will not queue up. 

    importer.stop()

    Description:

    Stops the polling. 
    importer.start()

    Description:

    Restarts the polling.

    Miso.Importer.GoogleSpreadsheet

    • key - The google spreadsheet key
    • worksheet - (optional) An optional worksheet id. Default is always first worksheet.

    Miso.Parser

    To specify a custom parser, during dataset instantiation set the parser property to the class name of the parser you want. Some parsers require some custom properties. This section will document the properties you can set that will either cause this parser to be selected or will be required by it to continue.

    Built in Parsers include:

    • Miso.Parsers.Strict
    • Miso.Parsers.Object
    • Miso.Parsers.Delimited
    • Miso.Parsers.GoogleSpreadsheet

    The base Miso.Parser structure is:

    Miso.Parsers.Strict

    • strict - Set this to true when initializing a dataset to indicate the default internal format should be used.

    Miso.Parsers.Object

    The object parser expects an array of objects where each key is the column name.

    • data - The local var containing the data to be parsed.

    Miso.Parsers.Delimited

    • delimited - The string that is delimiting the column values.

    Miso.Events

    A mixin module that adds syncing behavior methods to a dataset. Adds ability to bind, unbind and trigger events. Assume that ds is an instance that has syncable behavior.

    The events mixin was taken from Backbone.js. 

    ds.bind(ev, callback, context)

    Params:

  • ev - The name of the event to bind to
  • callback - The callback to call when the event is triggered
  • context - The context to bind the callback to.

    • Returns:

    The object being subscribed to.

    Description:

    Binds a callback to the dataset in question. 
    ds.unbind(ev, callback)

    Params:

  • ev - The name of the event to remove the bindings of.
  • callback - Optional. If specified, only the specific callback for this event will be removed. Otherwise all callbacks for the event name will be removed.

    • Returns:

    The object being unbound from.

    Description:

    Unbinds a particular event and/or a callback. 
    ds.trigger(eventName, *args)

    Params:

  • eventName - The name of the event that should be triggered.
  • *args - Any additional arguments that should be passed to the callback.

    • Returns:

    The object being unbound from.

    Description:

    Triggers a specific event on a dataset. Passes the arguments to it.
