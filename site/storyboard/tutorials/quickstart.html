Quick Start!

There are two ways to use Storyboard:

In the Browser

To get storyboard into your web development environment, follow the following steps:

Download the right build for your environment:

↓ development ↓ production (minified) ↓ development with dependencies ↓ production with dependencies (minified)

Include the appropriate script tags:

Start using Miso.Storyboard in your code!

You should now have access to the Miso.Storyboard object in your code.

In Node.js

Install appropriate package:

npm install miso.storyboard

Use the library in code:

var Miso = require("miso.storyboard");
var story = new Miso.Storyboard({
  //...
});

Dependencies

If you chose to include the production version without built in dependencies, you may need to include them yourself. Storyboard requires the following libraries:

  • LoDash 0.9.1
  • Underscore.deferred.js 0.2.0

Creating Your First Storyboard

First, we define a new Miso.Storyboard and set the following parameters on it:

Required:

  1. initial – What's the name of the initial scene we will kick off from?
  2. scenes – The actual scenes we want to be a part of this storyboard. See below for how one defines a scene.

Optional:

  1. context – What should 'this' be inside each scene? By default the storyboard or scene itself.
  2. Any other method or property!! You can define your own functions and properties here. They will be available to each other under the this object, but to each scene they will be available under this.parent.myCustomFunction().

Defining a Scene

A scene has the following key methods:

  1. enter – The function that is called when a scene is entered. You don't have to define it if you don't want to. By default, it will just be a passthrough. If you want to prevent the transition from happening, you can return false from this function, and it will cancel the transition and fail the deferred.
  2. exit – The function that is called when a scene is exiting. You don't have to define it if you don't want to. By default, it will just be a passthrough. If you want to prevent the transition from happening, you can return false from this function, and it will cancel the transition and fail the deferred.
